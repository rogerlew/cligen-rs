# Execution

Published source commit `320b6d5` initialized a new 20-minute authority. All
eleven planned assets passed local identity and declared executable intent.
Staging transferred 4.61 GB; the wheelhouse sustained 10.4 MB/s and the source
archive 10.2 MB/s. Remote byte/hash/mode verification passed before allocation.

Slurm job `1013865` started immediately on `node03` and completed in 242
seconds. Typed entry evidence established that Lemhi supplied
`LD_LIBRARY_PATH`, but not `PYTHONHOME` or `PYTHONPATH`, despite
`--export=NONE`. The wrapper recorded presence only, cleared all three,
reconstructed the frozen environment, and passed every Python, CUDA, Rust,
loader, offline-build, capacity, and supervisor gate.

The toolkit collected the exact evidence allowlist, authenticated the gate
receipt, verified job-local absence, removed the marker-bound durable root,
released the recovery reserve, and closed. No recovery allocation ran.

During the final configuration-level audit, the live Cargo-vendor archive hash
was compared to the candidate record and failed. The builder had regenerated
the gzip header timestamp while preserving byte-identical extracted content.
Because configuration identity is byte-level, the package holds and emits no
attestation. The cached archive was subsequently reconstructed to the frozen
candidate hash for a new package; this package's evidence remains unchanged.
