# A10M1/A10M2 handoff

A10M0 accepts both independent successor starts:

- A10M1 may freeze corpus identities, station roles, and development versus
  confirmation access rules without using Lemhi compute.
- A10M2 may execute the accepted Lemhi integration matrix without acquiring
  corpus series or selecting an A10 model.

The successors may run in either order. A10M3 remains blocked until both an
accepted A10M1 corpus/role terminal and `A10M2-COMPUTE-READY` exist. A hold in
either package is preserved and resolved within the applicable scientific or
operational boundary; it is not silently bypassed.
