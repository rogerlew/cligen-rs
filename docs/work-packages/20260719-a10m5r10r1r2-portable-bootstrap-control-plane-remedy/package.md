## Objective

Execute a fresh, coherent rerun of the complete A10M5R10 architecture
portfolio without changing its science, calendar, role matrix, or
job-local-capacity contract. This package corrects the A10M5R10R1R1
pre-runtime failure: Lemhi login nodes provide `/usr/bin/python3.11`, but the
compute image does not. Compute jobs therefore extract the already
toolkit-hash-verified portable CPython runtime with POSIX shell before invoking
any Python 3.11 diagnostic or inline check.

## Frozen execution remedy

Login-host submission admission continues to invoke `/usr/bin/python3.11` and
records its exact binary identity. On a compute node, `bootstrap_environment.sh`
uses only POSIX shell utilities to create job-local directories and extract
`runtime.tar.gz`. Only after `$runtime_root/bin/python3 --version` proves the
pinned Python 3.11 runtime may setup diagnostics or inline Python checks run.
All compute-side setup diagnostics use `$runtime_root/bin/python3`; the same
runtime creates the isolated environment that executes science.

The outer wrapper's post-supervisor finalizer is the sole exception. It runs
only after the supervised command is terminal and is deliberately compatible
with the known compute `/usr/bin/python3` (Python 3.6). It has no future
annotations or Python 3.7+ syntax and publishes minimal, authentic failed gate
evidence when portable-runtime extraction cannot begin or complete. It cannot
authorize science, admission, retry, or a passing gate.

The control role completes first. Candidate roles then run in the unchanged
five ordered waves of two: monthly residual, annual/monthly residual,
hierarchical joint-factor, climate-normal hierarchical state-space, and
physics-conditioned hierarchical adapter, each at K1 and K2. Within a wave,
the second role requires the first role's authenticated ready setup receipt;
the next wave requires both prior roles terminally observed with successful
cleanup. At most two candidate jobs are live and at most one bootstraps.

If a submitted role is exhausted and failed, the A10M5O1R2-hardened toolkit
atomically stops the remaining matrix, classifies only never-submitted roles as
`NOT_EXECUTED_UPSTREAM_FAILURE`, collects the exact sparse evidence surface,
proves cleanup, and closes. It does not submit doomed dependent roles or
fabricate admission, job, gate, or resource evidence.
