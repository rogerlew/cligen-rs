# A5d1 Planned Artifact Inventory

Status: `SCAFFOLDED-NO-EVIDENCE`

This directory intentionally contains no solver result, selected development
path, confirmation object/target, or A5d confirmation candidate score. The
package's `Planned artifacts` section is the execution inventory. Exact
versioned filenames may advance only through an explicit retained amendment;
superseded evidence is never overwritten.

Bulk daily libraries and solver certificates are reproducible execution
products, not automatically repository artifacts. Keep them in a hash-bound
workspace/cache when manifests and minimal fixtures are sufficient. If a
binary or compressed archive must be retained in Git, add and verify an
appropriate Git LFS rule before staging it; record `git check-attr`, pointer
identity, and `git lfs fsck` in the gate results. Do not commit a large object
through ordinary Git merely because it was generated.

Confirmation data and targets are forbidden in this tree. A5d1 may contain
only synthetic fixtures, the 17 explicitly exposed development-station
libraries and derivatives, strict contracts/tooling, review records, and the
terminal structural-feasibility decision.
