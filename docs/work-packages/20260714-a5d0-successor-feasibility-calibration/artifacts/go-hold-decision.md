# A5d0 Go/Hold Decision

Status: `HOLD-CONTRACT-INCOMPLETE`
Decision date: 2026-07-14
Decision authority: package execution pending operator disposition
Evidence mode: Derived and static

## Decision

A5d0 closes `EXECUTED-HOLD-CONTRACT-INCOMPLETE`. The whole-year/block thesis
is mathematically capable of reallocating variance in a constructed library,
but no single bounded repeat-safe, calendar-safe, common-prefix selector has
been specified or demonstrated across actual faithful development libraries.
The package therefore cannot freeze a candidate ID, coefficient grammar, or
implementation handoff.

Two independent secondary holds also prevent confirmation:

- `HOLD-EVALUATION-UNCALIBRATED`: observation-scaled G3/G4 measures,
  faithful-clone familywise calibration, bootstrap availability, and a numeric
  WEPP response criterion do not yet have executable frozen contracts.
- `HOLD-CONFIRMATION-CORPUS`: all 17 Daymet and eight GHCN records in the
  repository are exposed A5a/A5b evidence; no untouched qualified corpus is
  present.

These are cumulative blockers. Resolving the selector alone does not authorize
a candidate campaign.

## Evidence summary

- **Feasibility:** constructive fixture preserves daily first/second moments,
  changes zero physical daily values, and raises between-year variance 40% by
  reducing the within-year term. A constant-within-variance counterexample
  proves the property is not guaranteed for arbitrary libraries.
- **Dependence:** a stationary `rho = 0.6` kernel exactly produces lag-one
  covariance `rho Var`, but its same-block repeat probability is 0.6778.
- **Null calibration:** eight zero-failure trials leave a one-sided 95% upper
  false-failure bound of 0.3123; 59 independent zero-failure audit trials are
  required to put it below 0.05.
- **Station planning:** 28 stations balanced seven per primary regime with
  19 improvements gives sign-null probability 0.04358 and planning power
  0.86155 at true station-improvement probability 0.75. This does not finish
  composite/spatial power analysis.
- **Uncertainty:** accepted revision 3 had usable Gate 1 and Gate 4 corpus
  bootstrap aggregates in 221/2,000 and 8/2,000 resamples.
- **WEPP:** no numeric acceptance bound or valid observed-response reference is
  registered.
- **Exposure:** clean; no A5d confirmation candidate or target values accessed.
- **Compatibility:** no faithful code, public profile, default, schema, legacy
  `.par`, provenance, or typed-output change.

## Ordered corrective actions

1. **Selector feasibility package:** regenerate hash-bound faithful-off
   development libraries; implement the complete constrained-weight solver and
   bounded repeat-safe/calendar-safe path alternatives; require feasibility at
   every exposed development station.
2. **Evaluation calibration package:** create a new versioned A5d evaluation
   contract, run faithful-clone calibration with a defensible independence
   model, repair bootstrap availability, and pin a WEPP criterion/reference.
3. **Confirmation corpus package:** use 28 stations as a provisional balanced
   sign-test planning floor, determine the actual minimum with a
   calibration-only spatial/composite power analysis, then acquire and seal
   that corpus plus a new long-record sensitivity set using metadata-only
   qualification before target values are exposed.
4. **New freeze review:** only after all three packages pass, pin exactly one
   candidate version and dispatch a complete 30-/100-year climate and WEPP
   campaign.

The first action is item 1. Items 2 and 3 can proceed independently, but none
may generate an A5d confirmation candidate.

## Operator implication

This hold rejects no model family and changes no roadmap default. It records
that the lower-risk block-selector concept needs one more development proof
before it is safe to specify, while the evaluation and confirmation instruments
also need separate freezes. Proceeding directly to production coefficients or
a confirmation matrix would violate ADR-0004.
