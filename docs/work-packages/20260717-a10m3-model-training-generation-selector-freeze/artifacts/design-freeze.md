# A10M3 design freeze

The exact prospective contract is the combination of the two A10 revision-1
specifications, their three JSON Schemas, the two machine-readable contract
files, the candidate-blind calibration, and the executable reference module.

The architecture is a compact conditional neural state-space generator. It is
not an unrestricted transformer or diffusion search. Calendar and transferable
site descriptors condition a bounded latent recurrence. Precipitation uses a
hurdle plus positive-amount head; daily context and storm descriptors are joint
outputs. N0 is complete pooling; N1 differs only by zero-centered hierarchical
station/tile deviations. Candidate or confirmation identity is never a feature.

The pre-output screen contains exactly the 12 configuration IDs in the machine
contract; the two latent-32/depth-3 combinations per pooling class are excluded
rather than left implicit. No other configuration may run. Development data may order frozen entries
but cannot propose another. At most two entries per pooling class reach full
development and at most one per class reaches the three-seed replay.

Only M1 `candidate_fit` rows fit normalization or weights. Validation is used
only for early stopping and diagnostics. The inherited 32 development objects
remain development-only. The accepted v2 98-object transfer manifest is the
only staging authority; failed earlier Daymet manifests are prohibited. Hot
arid's absent eligible USCRN fit station remains visible and is not filled.
Daymet's official leap-year December 31 mask remains authoritative.

The primary score is candidate-blind: inherited null scales standardize each
mandatory metric, metrics average within seven families, and families average
equally. A regime needs material improvement against both B0 and B1 at both
horizons and four noninferiority guards. Missingness never helps. The runtime
classification uses unrounded warm ratios, with exact 5x WARN and exact 10x
FAIL. Cold start has only its separate absolute gate.

M3 used no cluster allocation and inspected no candidate or confirmation target
bytes. M4 may qualify the data loader, model forward/backward step, checkpoint
round-trip, stateless generation, CPU export, and resource telemetry on one L40
under the exact canonical configuration. It may correct implementation defects
before scored output with a recorded prospective amendment, but cannot alter
scientific thresholds, model family, grid ceiling, roles, or selector.
