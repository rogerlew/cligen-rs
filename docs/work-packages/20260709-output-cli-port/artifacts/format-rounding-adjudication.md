# FORMAT Output-Rounding Adjudication

Date: 2026-07-09
Evidence mode: **executional** — every claim below is from commands run
this session against the pinned reference toolchain (Homebrew GCC
14.2.0_1 gfortran, `-O0 -ffp-contract=off -fprotect-parens
-fno-fast-math`, linuxbrew x86_64, glibc 2.39, Xeon E5-2697 v2).

## Question adjudicated

What text does gfortran's `Fw.d` / `Iw` output editing produce for an
arbitrary REAL*4 value — exactly, at the byte level — so that
`crates/cligen/src/fortran_format.rs` can reproduce the `.cli` surface
without ever invoking a float-to-decimal library?

## Probe design

Two standalone Fortran probes (committed beside this artifact; built
with the pinned profile — probes are *drivers of the reference
runtime's formatted-output machinery*, not modified reference code;
`reference/` untouched):

- `fmtprobe.f` — reads REAL*4 bit patterns as Z8.8 hex on stdin,
  writes one line per value:
  `hex|F4.0|F4.1|F4.2|F5.1|F5.2|F6.2|F7.5|F8.5|F9.2` — the nine
  descriptors on the `.cli` surface (daily row 2000, header
  642/778/644/500/520/555). Negated inputs carry a `-` prefix on the
  hex field.
- `tieprobe.f` — the targeted tie-case driver (exact binary halves:
  0.25, 0.75, 2.5, 3.5, 0.0625, ± variants) used to settle the
  rounding rule before the sweep.

Probe input population (generated by the sweep harness, Ran):
- every REAL*4 value appearing in all 12 golden `.cli` files' source
  trajectories (captured tap values),
- dense sweeps across each descriptor's representable range including
  the width-overflow boundaries,
- exact binary halves at every decimal position d ∈ {0,1,2,5},
- negative zero, subnormals, and sign-boundary neighborhoods.

Total capture: **6,371,240 probe lines × 9 descriptors = 57,341,160
formatted fields** (455 MB, local evidence capture; SHA-256
`df8596f4e105fcfb5df156efea08e759b149e49b9c945b7e61c011d61a877777`).
The first 20,000 lines (180,000 fields) are committed at
`fixtures/taps/format/fmt-pairs-sample.txt` and gated in CI by
`tests/format_identity.rs::f_edit_matches_gfortran_sample`; the full
capture runs behind `#[ignore]` via `CLIGEN_FMT_SWEEP`.

## Adjudicated semantics (all verified over the full sweep, 0 mismatches)

1. **Rounding rule**: `Fw.d` rounds the **exact binary value** of the
   REAL*4 to `d` decimal places with **ties-to-even** (round-half-even).
   Verified on exact binary halves: `0.25 → 0.2`, `0.75 → 0.8`,
   `2.5 → "2."` (d=0), `3.5 → "4."`, `0.0625 → 0.06`. This rules out
   both ties-away ("printf" %.1f on doubles) and shortest-repr
   (Ryū/Grisu) semantics.
2. **d = 0** emits a trailing decimal point: `619.`, `  0.`.
3. **Negative zero** keeps its sign: `-0.0`.
4. **Width overflow, first relief**: the leading zero of `0.`/`-0.` is
   dropped to fit (`-.00`, `-.00017`).
5. **Width overflow, terminal**: the field fills with `w` asterisks.
6. `Iw` right-justifies; overflow fills with asterisks.

`f_edit` implements rule 1 by exact u128 integer arithmetic on the
decomposed (mantissa, exponent): `n = round_half_even(|v|·10^d)` via
integer quotient/remainder with a `2r` vs `den` comparison and
`q + (q & 1)` on equality — no floating-point rounding anywhere.

## Byte-level record semantics discovered during golden parity

Three record-level behaviors beyond field formatting, each found as a
first-divergent-line during the 12-golden iteration and pinned with a
source citation:

1. **Header command echo carries one trailing blank** — the main
   program's `av_len` scan sets the length one past the last non-blank
   character, so `arg_v(:av_len)` appends exactly one space after the
   echoed command (cligen.f:670-674, 678-682).
2. **A trailing X descriptor at end-of-record emits nothing** — X
   editing only advances the position; characters materialize only if
   later output lands at or past them. The final `1x` of format 500's
   `12(f5.1,1x)` group therefore produces no trailing blank
   (record ends at the highest written column).
3. **The run-end blank line is written only on normal termination** —
   the `write(7,*) ' '` at cligen.f:965-966 sits in the
   `moveto ≠ 10/230/225` branch; observed runs ending via the stop
   protocol (moveto = 225) close unit 7 without it (cligen.f:978).

## Verdict

`fortran_format::f_edit`/`i_edit` are **pinned platform-independent**:
the semantics are exact integer decimal rounding of the binary value,
not a reimplementation of any libc/libgfortran float-printing path, so
no §1.3 escalation hold applies. Acceptance evidence:

- **Ran**: full-sweep adjudication — 57,341,160 fields, **0
  mismatches** (`format_identity::f_edit_matches_gfortran_full_sweep`).
- **Ran**: 12/12 golden `.cli` files reproduced **byte-identically**
  from typed run inputs (`cli_parity::goldens_reproduced_byte_identically`).
