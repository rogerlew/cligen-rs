### Cross-reference interpretation (verification addendum, Claude Code, 2026-07-09)

Row-level comparison of the observed-mode production files against the
goldens shows the production `.cli` files are **not raw CLIGEN output**:
wind direction, wind velocity, radiation, and dew point carry different
*values* (e.g. Mt Wilson row 1: production `186 5.3 234 0.3` vs generator
`200. 3.8 232. -2.4`), and all numeric fields are requantized to one
decimal (`0.08 → 0.1`, `14.15 → 14.2`). This is wepppy's post-generation
rewrite (observed-variable substitution via the wepppyo3 round-trip), not
build-flag sensitivity and not merely formatting. Two consequences:

- The **stochastic** cross-references are raw CLIGEN output, so their
  byte-range identity with the pinned `-O0` deterministic build is a real
  measurement: on these trajectories the production build's
  reordering-permitted flags (`-O3 -fno-protect-parens` + fast-math
  family) changed nothing, on the same host/libm.
- The **observed** cross-references measure the wepppy round-trip, not
  the build; no raw-output flag-sensitivity instrument exists for
  observed mode in the vendored set. Incidentally, this documents that
  production observed climate files carry storm duration/`tp`/`ip` at
  reduced (one-decimal) precision — evidence for the A3 single-pass
  substitution goal.
