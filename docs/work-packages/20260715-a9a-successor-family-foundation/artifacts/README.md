# A9a foundation artifacts

These executed artifacts make the successor requirements independent of the
originating conversation:

- `evidence-to-requirements-ledger.md` traces accepted A5--A8 and SOTA evidence
  to A9 requirements;
- `model-family-envelope.md` defines the minimum joint model architecture;
- `tuning-harness-contract.md` defines tunable development and the locked
  confirmation firewall; and
- `data-and-evaluation-plan.md` defines source, regime, evidence-role, and
  climate-metric requirements;
- `authority-manifest-v1.json` hashes the exact accepted evidence set;
- `exposure-manifest-v1.json` hashes prior source records and freezes the 37
  exposed station IDs and 14 exposed model identifiers;
- `confirmation-metadata-selection-v1.json` freezes the metadata-only 18-site
  USCRN roster and records that no target series was accessed;
- `objective-registry-v1.json` defines engineering and climate objectives,
  availability, baseline-zero, uncertainty, and selection roles;
- `fixture-plan.md` defines the 20 synthetic/adverse fixture groups;
- `a9b-handoff.md` is the implementation-ready successor package contract;
- `capture-a9a-authorities.py` reproducibly rebuilds the authority/exposure
  manifests; and
- `verify-a9a.py`, `review.md`, and `gate-results.md` close the package.

They define a research foundation, not a fitted candidate or promotion
decision. No candidate coefficients, target climate, accepted runtime IDs, or
production behavior are present.
