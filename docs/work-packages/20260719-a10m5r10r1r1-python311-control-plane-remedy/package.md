## Objective

Execute a fresh, coherent rerun of the complete A10M5R10 architecture
portfolio without changing its scientific or job-local-capacity contract. This
package corrects the A10M5R10R1 pre-runtime failure by pinning every remote
host/control-plane Python invocation to `/usr/bin/python3.11`; the extracted
portable runtime remains the sole interpreter for environment creation and
scientific execution.

## Predecessor evidence and frozen science

A10M5R10R1 control job `1014042` authenticated its submission admission and
then failed in one second because the wrapper invoked `/usr/bin/python3`
(Python 3.6) for a source file requiring Python 3.11. No runtime archive,
wheelhouse, corpus, or scientific process was opened. Its ten candidate roles
also ended before runtime extraction, only to satisfy the historical toolkit's
matrix-settlement requirement. That run remains
`HOLD-A10M5R10R1-PYTHON311-CONTROL-PLANE` and carries no architecture
interpretation.

This package authenticates the R1 operational summary, disposition, resource

`artifacts/predecessor-evidence-identities.json`. It authenticates the original
A10M5R10 portfolio contract and all twelve direct/transitive science
dependencies byte-for-byte under
`artifacts/science-dependency-identities.json`. Results from either prior run
cannot substitute for a successor role.

The package retains the complete 1,440-object calendar/missingness scan from
the unchanged R0 science source. `artifacts/calendar-preflight.json` binds the

## Frozen execution remedy

The control role completes first. Candidate roles then run in the same five
ordered waves of two: monthly residual, annual/monthly residual, hierarchical
joint-factor, climate-normal hierarchical state-space, and
physics-conditioned hierarchical adapter, each at K1 and K2.

Within a wave, the first role is submitted alone. The second role may be
submitted only after the first publishes authenticated `setup.json` with
successful installation/check, deleted wheelhouse and pip cache, and
`ready_for_science: true`. The next wave starts only after both prior jobs are
terminal, observed, and show successful exact job-local cleanup. At most two
candidate jobs are live and at most one is bootstrapping.

Every pre-runtime/control-plane Python command in the staged remote job sources
uses the absolute `/usr/bin/python3.11` path. The admission protocol also uses
that path on the login host. `bootstrap_environment.sh` records and gates the
host interpreter version before opening runtime assets. Commands using
`$runtime_root/bin/python3` are intentionally preserved: that registered
portable CPython 3.11 runtime creates the environment and runs science.

If a submitted role is exhausted and failed, the A10M5O1R2-hardened toolkit
may atomically stop the whole remaining matrix, classify only never-submitted
roles as `NOT_EXECUTED_UPSTREAM_FAILURE`, collect the exact sparse evidence
surface, prove cleanup, and close. It must not submit doomed dependent roles or
fabricate admission/job evidence.
