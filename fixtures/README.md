# Fixture Station Set

Four real station cases selected by the operator (2026-07-09) from
production WEPPcloud runs, spanning both hemispheres, both generator
modes, and the observed end-of-record edge case. Vendored inputs +
production cross-references; **goldens are not vendored here** — they are
generated by the golden-fixture-harness package from the pinned reference
build (ADR-0001 §4).

| Fixture | Station | Lat | Mode | Inputs |
|---|---|---|---|---|
| `new-meadows-id/` | NEW MEADOWS RS, Idaho (`id106388.par`) | 44.97 | Stochastic generation | `.par` + `wepp.inp` stdin answers |
| `mt-wilson-ca/` | MT WILSON FC 338B, California (`ca046006.par`) | 34.23 | Observed (`-O`), DAYMET 1990–2010 | `.par` + `ws.prn` |
| `jeogla-au/` | Jeogla Station, Australia (`ASN00057011.par`, GHCN) | −30.58 | Stochastic generation | `.par` + `wepp.inp` stdin answers |
| `fish-springs-ut/` | FISH SPRINGS REFUGE, Utah (`ut422852.par`) | 39.83 | Observed (`-O`), gridMET 2020–2026 | `.par` + `ws.prn` |

Coverage rationale:

- **Latitude spread including southern hemisphere** — the storm-intensity
  machinery is latitude-responsive (Bofu Yu `alphb`), and Jeogla exercises
  the seasonal inversion.
- **Both live modes** — stochastic generation and observed (`-O`).
- **End-of-record edge**: the Fish Springs `ws.prn` runs to 2026-12-31
  with trailing `9999` sentinel rows (the record ends mid-2026). The
  mechanics, adjudicated from source (2026-07-09):
  - `9999` is a per-field "generate this value" instruction to CLIGEN —
    `day_gen` sets `nsim=1` (simulate precip) / `msim=1` (simulate temps)
    per 9999 field (`cligen.f:3076-3079`) — so a sentinel-padded final
    year yields a **complete** year in the `.cli` with the unobserved
    tail stochastically filled.
  - A hard-truncated `.prn` (no padding) ends the `.cli` **mid-year**:
    post-5.323 CLIGEN terminates cleanly at EOF; pre-5.323 it appended a
    bogus extra day (the 5.323 fix).
  - The padding is **workflow-necessary because of WEPP**, not CLIGEN:
    WEPP's `stmget` reads climate in 10-record chunks
    (`read(13,*,end=20)`, `stmget.for:121` in the wepp-forest baseline)
    and a mid-data EOF falls through to stale array contents — a run
    configured through the final year against a short `.cli` silently
    recycles the previous chunk's storm records rather than erroring.
    wepppy therefore pads by default (`df_to_prn(pad_to_end_of_year=True)`,
    `wepppy/climates/cligen/cligen.py`).
  - Phase B still crafts the hard-truncated `.prn` variant to pin
    CLIGEN's post-5.323 EOF behavior (partial-year emission) as a
    faithful-mode fixture; the padded case pins the sentinel-fill path.
    Both behaviors are port surface. (The A3 observed-parquet input
    should make end-of-record semantics typed and explicit — this
    sentinel archaeology is the argument for it.)

## Provenance

- Source runs (operator's production WEPPcloud instance, copied
  2026-07-09): `ungenerous-caustic`, `strained-mod`, `invalid-winter`,
  `hydrodynamic-discomfort`.
- All four `wepp.cli` files were produced by **CLIGEN VERSION 5.32300** —
  the same source version vendored at `reference/cligen532/` — by the
  WEPPcloud production build (compiler/libm provenance unknown).
- `.par` files: USDA/GHCN station parameter files as localized by wepppy
  (public data). `ws.prn` observed series derive from DAYMET (Mt Wilson)
  and gridMET (Fish Springs), both public datasets.
- `cligen_wepp.log`: the production invocation logs, verbatim.

## Role of the vendored `wepp.cli` files

They are **production cross-references, not goldens**: same source
version, unknown build. Comparing them against the pinned reference
build's goldens in Phase A measures build/libm/flag sensitivity for free —
byte-identity would suggest low libm sensitivity; divergence localizes it
(first-divergent-day differ). Goldens for port acceptance come only from
the provenance-pinned reference build.

Not vendored (deliberately): `wepp_cli.parquet` (wepppy's parquet
rendering — prior art for SPEC-CLI-PARQUET, retrievable from wepppy) and
`atlas14_intensity_pds_mean_metric.csv` (derived analysis; relevant to the
A5 NOAA storm-curve work, noted here for the trail).
