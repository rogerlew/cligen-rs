# Fixture Station Set

Four real station cases selected by the operator (2026-07-09) from
production WEPPcloud runs, spanning both hemispheres, both generator
modes, and the observed end-of-record edge case. Vendored inputs +
production cross-references; **goldens are not vendored here** ‚Äî they are
generated by the golden-fixture-harness package from the pinned reference
build (ADR-0001 ¬ß4).

Coverage rationale:

- **Latitude spread including southern hemisphere** ‚Äî the storm-intensity
  machinery is latitude-responsive (Bofu Yu `alphb`), and Jeogla exercises
  the seasonal inversion.
- **Both live modes** ‚Äî stochastic generation and observed (`-O`).
- **End-of-record edge**: the Fish Springs `ws.prn` runs to 2026-12-31
  with trailing `9999` missing-data sentinel rows (the record ends
  mid-2026) ‚Äî the end-of-record class behind the 5.323 `day_gen` fix.
  Phase B of the harness package should additionally craft a
  hard-truncated `.prn` variant (file ends mid-year without sentinel
  padding) to pin the exact 5.323 EOF shape.

## Provenance

- Source runs (operator's production WEPPcloud instance, copied
  2026-07-09): `ungenerous-caustic`, `strained-mod`, `invalid-winter`,
  `hydrodynamic-discomfort`.
- All four `wepp.cli` files were produced by **CLIGEN VERSION 5.32300** ‚Äî
  the same source version vendored at `reference/cligen532/` ‚Äî by the
  WEPPcloud production build (compiler/libm provenance unknown).
- `.par` files: USDA/GHCN station parameter files as localized by wepppy
  (public data). `ws.prn` observed series derive from DAYMET (Mt Wilson)
  and gridMET (Fish Springs), both public datasets.
- `cligen_wepp.log`: the production invocation logs, verbatim.

## Role of the vendored `wepp.cli` files

They are **production cross-references, not goldens**: same source
version, unknown build. Comparing them against the pinned reference
build's goldens in Phase A measures build/libm/flag sensitivity for free ‚Äî
byte-identity would suggest low libm sensitivity; divergence localizes it
(first-divergent-day differ). Goldens for port acceptance come only from
the provenance-pinned reference build.

Not vendored (deliberately): `wepp_cli.parquet` (wepppy's parquet
rendering ‚Äî prior art for SPEC-CLI-PARQUET, retrievable from wepppy) and
`atlas14_intensity_pds_mean_metric.csv` (derived analysis; relevant to the
A5 NOAA storm-curve work, noted here for the trail).
